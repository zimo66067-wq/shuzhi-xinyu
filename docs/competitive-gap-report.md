## 0. 先纠正简报里三个"想当然"（诚实优先，否则后面全是错靶）

通读代码后，简报中四个高危项里有两个的**naive 版本已经被代码规避**，必须据实修正，否则在评审会上用错误前提攻击，反被对方一句话化解：

| 简报假设 | 代码实际 | 结论 |
|---|---|---|
| STT 用 Web Speech API，大陆不可用 | 实际走 `MediaRecorder(webm) → /api/stt → ffmpeg → 腾讯云 ASR`（见 `frontend/src/lib/stt.js`、README 第 39 行） | naive 风险**已规避**；但残留风险换了形态，仍是致命级（见 2.3.1） |
| MediaPipe 依赖 Google CDN，大陆被墙 | 资源已全本地托管 `public/mediapipe/wasm` + `face_landmarker.task`，`useGLTF(...,false,false)` 关掉 draco/meshopt，全前端零外网 CDN 引用（grep 验证） | 风险**已主动规避**，反而是亮点；残留是"未在真机大陆网络验证" |
| 医疗器械监管定位 | README **标题**自称"自闭症儿童社交评估系统"，又贴"非临床"免责 | 风险**真实且更尖锐**：自相矛盾比纯做评估更容易被两头打 |
| 未成年人数据合规 | 有 PRIVACY.md（纸面较完整），但 app 内**无知情同意流程**，对话原文仍转发 DeepSeek | 风险**真实**，纸面合规 ≠ 落地合规 |

把这两个"已规避"项写进报告时，攻击点要转移到**残留风险**，而不是继续打已经被堵的洞——否则显得没读代码。

## 1. 执行摘要 —— 评委最可能当场毙掉项目的 Top 5 致命问题

> 这 5 条任意一条单独拎出来，都足以让一个临床专家或合规评委在 Q&A 环节让团队下不来台。

### ① 临床效度零证据链：5 维评分是自创的，映射不到任何循证量表
`prompt_scoring.txt` 的五维（主动发起/话题维持/情绪识别/礼貌用语/参与度）和 1-5 分锚点，**没有任何一处引用 ABA / ESDM / PEERS / SRS-2 / Vineland / ATEC**。这是一个 LLM 一次性打的分，无信度（同一段对话两次打分会漂）、无效度（凭什么这 5 维=社交能力）、无常模（5 岁的 60 分和 9 岁的 60 分不可比）。临床评委第一刀：**"你这个分数代表什么？拿什么证明它有意义？"** —— 团队答不上来。

### ② 监管定位自相矛盾：又叫"评估系统"，又喊"非临床"
README 标题写死"AI 数字人自闭症儿童社交评估系统"，家长后台叫"5 维社交能力评估"，同时又到处贴"仅演示·非临床""不能诊断"。在中国，**面向"自闭症儿童"做"评估/评分"**极易被认定为第二类医疗器械软件（SaMD）或医疗广告违规；而"非临床"免责又自我否定了产品的核心卖点。**两头不讨好**：要么是没注册的医疗器械（违规），要么是"自己都说不算数"的玩具（没价值）。

### ③ 危机处理是"静默降级"，没把真实成人实时拉进来
声学检测到 cry/agitated 时，代码只做了一件事：`setShowComfort(true)` 弹一条"要去看看小工具吗"的横幅（`ChatPage.jsx:157-167`）。**没有任何机制把家长/监护人实时叫到现场**，也没有求助热线、没有"我现在很不好"的一键呼叫。对一个可能正在崩溃（meltdown）的自闭症儿童，系统的反应是"建议你去看小动画"——这在临床和伦理上**不合格**，且一旦出事是直接法律责任。

### ④ 未成年人极敏感数据，明文本地存 + 出境第三方训练，且无知情同意
孩子姓名、年龄、完整对话、声学摘要明文存 localStorage（任何拿到设备的人可读）；对话原文（含姓名年龄）实时转发 DeepSeek，**按 DeepSeek 隐私政策可用于模型训练**（PRIVACY.md 第 69 行自己承认）。而 app 内**根本没有监护人知情同意勾选**——PRIVACY.md 第 81 行写的是"建议加上"，即未实现。《个人信息保护法》对不满 14 周岁未成年人要求**单独同意 + 专门处理规则**，这里一条没落地。

### ⑤ 没有纵向追踪、没有专业方 console = 机构永远不会续费
评分对比靠后端进程内的 `_last_score_cache / _prev_score_cache`（ARCHITECTURE.md 第 116 行），**重启即丢、换设备即丢**。没有跨周/跨月曲线、没有 IEP 式目标设定、没有给治疗师/特教老师的工作台。康复中心/特校买单的核心理由是"能看到孩子三个月的进步并写进个别化教育计划"——这个产品一次性出个报告就结束了，**机构试用完不会续费**，商业模式落不了地。

## 2. 分维度差距分析

> 每条差距五要素：【现状】【成熟产品标准】【严重度】【会被怎样攻击】【修复方向（可执行下一步）】

### 2.1 临床与监管

#### 2.1.1 评分维度自创，无循证框架映射
- **现状**：`prompt_scoring.txt` 五维 + 1-5 锚点为团队自定义，无文献/量表来源；分数由 `deepseek-chat` 单次生成，后端只做 JSON 修复（`_validate_and_repair_score`），不做信度控制。
- **成熟产品标准**：严肃的 ASD 社交训练/评估产品会把指标显式锚定到公开框架（如 PEERS 的对话技能清单、SRS-2 的社交反应维度、Vineland-3 沟通/社交子域、ATEC 量表），并披露评分的信效度来源或注明"参考性启发式"。
- **严重度：致命**
- **会被怎样攻击**："这五个维度的理论依据是什么？和 SRS-2 怎么对应？同一段对话让模型打两遍分一样吗（信度）？没有常模，6 岁和 10 岁的分数怎么比？" —— 现状无一能答。
- **修复方向**：① 立即在 prompt 和报告 UI 标注"本评分为参考性启发式，非标准化量表，不可用于诊断/分级"；② 把五维显式映射到一个公开框架的子条目（推荐 PEERS 对话技能 + Vineland 社交沟通子域），在 `docs/` 写一份《评分维度映射说明》列出每一维对应的框架条目与文献；③ 做一个最小信度自检：同一对话调用评分 3 次，计算维度方差，方差大的维度在 prompt 里收紧锚点或降权。

#### 2.1.2 产品定位在"评估/诊断"红线上反复横跳
- **现状**：README 标题"…自闭症儿童社交评估系统"、家长后台"5 维社交能力评估"，与多处"仅演示·非临床""不能诊断"并存（`ParentPage.jsx:250`、`StartupPage.jsx:176`、README 第 14-20 行）。
- **成熟产品标准**：合规的同类产品会把自己**清晰锁定在"训练辅助/陪伴/观察记录工具"**，全程不用"评估/筛查/诊断"作为对外定位，避免触发《医疗器械监督管理条例》对 SaMD 的注册要求和医疗广告法。
- **严重度：致命**
- **会被怎样攻击**："你面向自闭症儿童做评分，这是不是医疗器械？注册证呢？没有就是违规上市。" 或反过来："你自己说非临床，那家长凭什么信这个分数？"
- **修复方向**：① 全局把对外措辞从"评估系统/社交能力评估"改为"社交练习陪伴工具 / 家长观察记录"，README 标题先改（这是评委最先看到的一行）；② 评分模块对家长改叫"本次练习回顾"，分数旁固定标注"非诊断、非分级，仅供家长了解今天的参与情况"；③ 走机构试点路线（康复中心/特校）而非 C 端，在机构语境下定位为"治疗师的辅助记录工具"，由持证治疗师解读，规避 C 端直接出分的监管风险。

#### 2.1.3 危机升级是静默的，未把真实成人实时拉入
- **现状**：cry/agitated → 弹"去工具盒"横幅；silence → 字幕"心屿在这里"；15 分钟 → 弹"也找朋友玩玩吧"。**全程无家长实时告警、无求助通道**（`ChatPage.jsx:157-167`、`useAcousticAnalysis.js`）。prompt 里"绝不提供医疗/诊断建议"做了，但"把成人拉进来"没做。
- **成熟产品标准**：面向脆弱未成年人的产品在检测到 distress 时，至少要有：① 屏幕醒目提示"请让身边的大人来帮帮你"；② 可配置的家长实时通知（本地声音/震动/推送）；③ 危机求助资源入口（如全国心理援助热线）。
- **严重度：致命**
- **会被怎样攻击**："孩子情绪崩溃时你的系统让他去看深呼吸动画？真实成人在哪？出了事谁负责？" —— 这是伦理 + 法律双杀。
- **修复方向**：① 在 cry/agitated 触发时，横幅文案改为明确指向真实成人（"我们去找爸爸妈妈好吗？"）并配一个醒目的"叫大人"按钮，点击播放提示音把家长引到现场；② 设计"必须有监护人在场"为硬前提，启动页要求勾选"我（监护人）会全程陪同"；③ 在工具盒或家长后台放置危机求助资源（地区心理援助热线），并在 README/PRIVACY 写明"本产品不是危机干预工具，紧急情况请联系专业机构"。

#### 2.1.4 prompt 安全层依赖单模型自觉，无确定性兜底
- **现状**：不诊断、不索取家庭信息、负面情绪降级，全部写在 `prompt_chat.txt` 里靠模型遵守；后端有注入检测（README 第 50 行）但对"模型自己说出诊断性/索取隐私的话"无输出侧硬过滤（report 有 BLOCKED_WORDS，chat 回复没有同等过滤）。
- **成熟产品标准**：儿童安全关键约束应有"prompt + 输出侧确定性过滤"双保险，模型越权输出（如说出"你这是自闭症""你家住哪"）应被规则层拦截。
- **严重度：严重**
- **会被怎样攻击**："模型哪天抽风说出'你可能有病'或问'你家在哪'，你拿什么拦？全靠它听话？"
- **修复方向**：把 report 已有的 `_filter_report_text` / BLOCKED_WORDS 思路下沉到 `/api/chat` 的回复出口，对 chat reply 也做一遍污名词 + 隐私索取词（住址/学校/电话）正则过滤，命中则替换为安全兜底句。

### 2.2 数据合规与隐私

#### 2.2.1 app 内无监护人知情同意流程（仅纸面建议）
- **现状**：`StartupPage.jsx` 只采集名字 + 年龄即进入对话，无任何同意勾选；PRIVACY.md 第 81 行明确写"建议在启动页面加上同意勾选项"——即承认未实现。
- **成熟产品标准**：处理 <14 岁未成年人个人信息，《个人信息保护法》第 31 条要求取得监护人**单独同意**并制定专门处理规则；落地形态是首次使用强制弹出同意书 + 勾选 + 可查看隐私政策。
- **严重度：致命**
- **会被怎样攻击**："孩子姓名、对话内容都是个人信息，你处理未成年人信息的单独同意在哪？专门处理规则在哪？" —— 直接踩 PIPL。
- **修复方向**：① 首次启动（`isFirstSetup`）插入一屏监护人同意页：列明采集项（名字/年龄/对话/声学）、用途、第三方流向（DeepSeek/腾讯云）、留存与删除方式，必须勾选"我是监护人且同意"才能继续；② 把同意状态与时间戳存 localStorage，未同意不放行；③ 在同意页一句话点明"对话会发送给第三方 AI 服务用于生成回复"。

#### 2.2.2 极敏感数据明文落地，且出境第三方可用于训练
- **现状**：`xinyu_childInfo / xinyu_messages / xinyu_acousticHistory` 明文存 localStorage（`App.jsx`）；对话原文转发 DeepSeek，PRIVACY.md 第 69 行承认第三方"可能用于模型训练"。
- **成熟产品标准**：自闭症未成年人对话属敏感个人信息，应做数据最小化、本地加密或不落地、与第三方签 DPA 并要求"不用于训练"，能力允许时优先私有化/本地推理。
- **严重度：严重**
- **会被怎样攻击**："自闭症孩子的对话被拿去训练大模型，家长知道吗？同意了吗？这数据要是泄露，是给一个孩子贴了终身标签。"
- **修复方向**：① 短期——在同意书显著披露"用于训练"事实，让家长知情；并探索 DeepSeek/腾讯云是否提供"不用于训练"的企业条款，谈下来后更新 PRIVACY；② 中期——messages 写 localStorage 前做对称加密（家长密码派生 key），降低设备被他人翻看的风险；③ 把"留存周期/删除权"从"手动清 localStorage"升级为家长后台一键"导出 + 永久删除本设备数据"按钮。

#### 2.2.3 多模态采集存在过度之嫌（面部 + 声学）
- **现状**：家长视图启用摄像头跑 MediaPipe 抽微笑/皱眉/张嘴/正脸（`FaceMetrics.jsx`，本地处理不落盘，已是优点）；声学摘要落 localStorage 并在家长后台展示"音高单调可能与社交沟通模式相关"（`ParentPage.jsx:397`）。
- **成熟产品标准**：对儿童做面部表情分析属高敏生物特征类处理，需单独同意与必要性论证；声学"诊断性"解读措辞要克制。
- **严重度：一般**
- **会被怎样攻击**："你给一个自闭症孩子做人脸表情分析？必要性论证呢？单独同意呢？""音高单调=社交沟通模式，这不是变相诊断？"
- **修复方向**：① 面部分析默认关闭已做对（visible 才启），再补一句使用前的单独提示"即将开启摄像头做面部表情参考，仅本地处理不上传，是否继续"；② 家长后台声学解读把"可能与社交沟通模式相关"改为更中性的"今天说话的音量/停顿情况"，去掉任何指向"模式/障碍"的暗示。

### 2.3 大陆落地技术风险

#### 2.3.1 语音链路虽用国内 ASR，但仍是 demo 能否跑起来的最大单点
- **现状**：STT=腾讯云一句话识别（`SentenceRecognition`，单次、非流式、上限 ~60s）；getUserMedia 强制 HTTPS；生产部署"脚本就绪、尚未真实上线"（README 第 165 行），且阿里云大陆主机未备案无法用域名/HTTPS。
- **成熟产品标准**：语音为核心交互的产品应有：流式 ASR（边说边出字）、弱网重试、移动端（尤其 iOS Safari）编码兼容、以及一个**确定能跑的 HTTPS 演示地址**。
- **严重度：致命（对 demo 而言）**
- **会被怎样攻击**："现场连你的线上地址我试试语音"——没有备案的 HTTPS 地址就只能本机 localhost 演示；"用 iPhone 演示"——iOS Safari 的 MediaRecorder 历史上不出 webm/opus，`pickSupportedMimeType` 会回落 mp4，后端 ffmpeg 转码路径是否覆盖 mp4 未验证。
- **修复方向**：① 比赛前务必准备一条**可用的 HTTPS 演示链路**：用香港轻量服务器（DEPLOYMENT.md 第 7 节，免备案）或临时内网穿透 + 自签/临时证书，先保证现场能连；② 真机过一遍 iOS Safari：确认录到的 mime 类型，并在 `stt_service.py` 的转码侧确认 mp4/m4a 可解；③ 文档里写清"当前为一句话识别（按住说话），流式 ASR 为后续项"，主动管理评委预期，别让对方先发现。

#### 2.3.2 弱网/断网鲁棒性：机构网络差，云端 LLM 一抖就哑
- **现状**：有全局 online/offline 横幅（`App.jsx`），断网时禁用输入；但对话/评分/报告**全部强依赖云端 DeepSeek**，DEPLOYMENT.md 第 185 行已承认"国内→DeepSeek 偶发 SSL EOF"，且无重试。
- **成熟产品标准**：面向康复中心/特校（网络普遍不稳）的产品应有请求重试 + 超时降级 + 关键路径离线兜底（哪怕是固定脚本对话）。
- **严重度：严重**
- **会被怎样攻击**："我们机房网络很差，你这一卡，正在情绪上头的孩子等十秒没反应，现场就失控了。"
- **修复方向**：① `llm_client` 加指数退避重试（2-3 次）+ 总超时，超时返回温和兜底句（已有兜底字符串，串进重试逻辑即可）；② 设计一个"离线陪伴模式"：网络不可用时切到一组预置的本地安全话术（打招呼/安抚/告别），保证孩子永远有回应；③ TTS 失败已会降级 Web Speech（`tts.js`），把这种降级思路在 chat 路径也补齐。

#### 2.3.3 Google CDN 风险已规避，但缺"真机大陆网络"验证证据
- **现状**：MediaPipe 本地托管、draco/meshopt 关闭、前端零外网 CDN（已 grep 验证）——这是**做对了的**。但 README 自承部署未真实上线，没有"在大陆网络真机跑通、无任何 timeout 请求"的证据。
- **成熟产品标准**：声称"无外网依赖"要有一次真机大陆网络的网络面板截图佐证（DevTools Network 无 gstatic/googleapis/失败请求）。
- **严重度：一般**
- **会被怎样攻击**："你说没墙问题，证据呢？" —— 现在只能口头保证。
- **修复方向**：部署到香港/已备案主机后，开一次 DevTools Network 跑完整流程，截图存档放进 `docs/`，作为"大陆可用"的硬证据；顺带 CI 里加一条 grep 断言防止有人未来重新引入外网 CDN。

### 2.4 产品深度

#### 2.4.1 谱系内"千人一面"：有年龄分档，无能力/表现自适应
- **现状**：按年龄分 3 档（low/mid/high）调字号、表情数、句长（`getAgeBand`、`AGE_BAND_PROMPTS`）——这是按"岁数"，不是按"能力"。同年龄、不同语言能力的孩子拿到完全一样的话术；难度不会因孩子表现自动升降。
- **成熟产品标准**：ASD 是谱系，成熟训练产品按孩子的**当前表现**做难度自适应（回应短→降难度给句式选项；持续投入→升难度），而非只按年龄一刀切。
- **严重度：严重**
- **会被怎样攻击**："一个 8 岁极简语 minimal verbal 的孩子和一个 8 岁高功能的孩子，你给的是同一套？这叫个性化？"
- **修复方向**：① 用已有的轮次/回应长度/声学摘要做一个轻量难度信号：连续 N 轮回应≤2 字 → 降档（强制给句式选项、缩短句长）；持续多轮有内容 → 升档；② 把这个动态档位作为新占位符注入 chat prompt（复用 age_band 的注入机制）；③ 首次设置时让家长粗选孩子语言水平（无语言/单词/短句/成句）作为初始难度，比单纯年龄更准。

#### 2.4.2 多模态支持薄弱：缺视觉支持 / PECS / 结构化轮流提示
- **现状**：交互=文字 + 语音 + emoji 贴纸（`EMOJIS_*`）。无图片/图卡选择（PECS 式）、无可视化轮流提示（whose turn）、无视觉日程。
- **成熟产品标准**：真实社交训练大量依赖视觉支持（图卡、社交故事、轮流条），尤其对低语言能力孩子，图片交换比纯文本有效得多。
- **严重度：严重**
- **会被怎样攻击**："自闭症干预的黄金实践是视觉支持和 PECS，你一个以语言对话为唯一主轴的产品，覆盖不了谱系里语言弱的那一大半。"
- **修复方向**：① 给 emoji 贴纸升级为"图卡作答"：在 scenario 里把"我想要 X 面包"等目标句做成 2-3 张图片选项让孩子点选，而非必须说/打字；② 加一个极简"轮到谁"视觉条（心屿说话时高亮心屿，等孩子时高亮孩子头像），降低轮流规则的认知负担；③ 这两项都能直接用现有 scenario 状态机驱动，改动集中在前端 UI。

#### 2.4.3 场景库太薄：只有一个超市，且为线性脚本
- **现状**：仅 `prompt_scenario_supermarket.txt` 一个场景，5 子场景写死线性顺序（enter→find→ask_price→pay→leave），分支容错有限。
- **成熟产品标准**：成熟产品有厚场景库（学校、医院、餐厅、操场冲突、借还东西…）且每个场景有多分支与泛化练习。
- **严重度：严重**
- **会被怎样攻击**："就一个买面包？社交泛化怎么办？孩子在超市学会了，到学校还是不会。"
- **修复方向**：① 把场景做成数据驱动：抽象出"子场景目标 + 角色 + 提示句"的 schema，新增场景=加一个 prompt + 一条 `SCENARIO_PROMPTS` 字典项 + 前端一个卡片（ARCHITECTURE.md 第 189 行已规划，落地它）；② 比赛前至少补到 3 个场景（超市/学校问好/餐厅点餐），数量比深度更能撑住"场景库"叙事；③ 每个场景结束后给一句"在家也可以这样练"的泛化建议（复用 next_training 字段）。

### 2.5 训练闭环与纵向追踪

#### 2.5.1 只有单次报告，无跨周/跨月进步追踪
- **现状**：评分进步对比靠后端进程内缓存 `_last/_prev_score_cache`（ARCHITECTURE.md 第 116 行、第 193 行明说加 SQLite 会"违反无状态承诺"），重启/换设备即丢；前端不存历史评分。
- **成熟产品标准**：训练类产品的核心价值在纵向——周/月进步曲线、维度趋势、里程碑，让家长和机构看到"练了有用"。
- **严重度：致命（对留存/续费而言）**
- **会被怎样攻击**："家长练了三个月，想看孩子进步曲线，你给不出来？那他为什么继续用？"
- **修复方向**：① 评分生成时把 `{date, total_score, dimensions}` 追加存一条历史（先 localStorage 起步，够撑 demo 和家庭场景）；② 家长后台加一张"近 N 次总分 + 维度趋势"折线（Recharts 已在用，零新依赖）；③ 机构版再考虑后端持久化（SQLite + 同步更新 PRIVACY 的留存说明）。

#### 2.5.2 无目标设定（IEP 式），训练没有方向感
- **现状**：每次评分给 `improvements / next_training` 建议，但**没有可勾选/可追踪的训练目标**，建议看完即弃，不形成"设定目标→多次练习→达成"的闭环。
- **成熟产品标准**：对接特教的产品会让治疗师/家长设定个别化目标（如"两周内主动打招呼 3 次"）并追踪达成率，对齐 IEP。
- **严重度：严重**
- **会被怎样攻击**："你的训练没有目标，就是每天随便聊聊打个分，和 IEP 怎么衔接？特教老师拿它干嘛？"
- **修复方向**：① 把 `next_training` 升级为可"采纳为目标"的对象：家长点"设为本周目标"后存下来，之后每次报告显示该目标的相关维度变化；② 目标用结构化模板（行为 + 频次 + 周期），方便日后导出对接 IEP；③ demo 里展示"目标卡 + 达成进度条"即可讲出闭环故事。

#### 2.5.3 没有给治疗师/老师的专业方 console（机构落地入口缺失）
- **现状**：只有家长后台（密码门 + 单次评分 + 报告），无多孩子管理、无治疗师视角、无批注/干预记录。
- **成熟产品标准**：机构落地的真正入口是治疗师工作台——管理一批孩子、查看各自纵向数据、写干预记录、导出给家长。
- **严重度：严重**
- **会被怎样攻击**："康复中心一个治疗师带十几个孩子，你的'家长后台'怎么用？机构凭什么采购一个只服务单个家庭的工具？"
- **修复方向**：① 规划"治疗师 console"角色（多孩子列表 + 每个孩子的纵向曲线 + 干预备注），先出原型图放进 `docs/` 讲商业故事，不必全实现；② 数据模型上把 childInfo 从"单个"改为"可多个 + 选择当前"，为多孩子管理留接口；③ 机构版定价/采购故事围绕这个 console 展开（见 2.6）。

### 2.6 比赛与商业

#### 2.6.1 P0 外功其实做了，但 demo 的阿喀琉斯之踵是"上不了线 + 定位踩雷"
- **现状**：TTS/STT/3D（VRM）均已实现（README 核心功能表全 ✅），所以简报担心的"P0 空壳"**不成立**——外功在。但：VRM 模型不入仓（18MB 外部分发），生产未上线（未备案），现场只能 localhost 演示；叠加 2.1.2 的"评估系统"定位雷。
- **成熟产品标准**：比赛 demo 要有稳定可连的线上地址 + 清晰不踩雷的一句话定位 + 3 分钟讲清"问题-方案-效果"的脚本。
- **严重度：致命（对拿奖而言）**
- **会被怎样攻击**："给个链接我们会后体验"——拿不出；"你到底是评估还是陪伴？"——定位含糊被追问。
- **修复方向**：① 备一份**离线/本机 demo 脚本**（localhost 麦克风可用），同时尽量上一个香港免备案 HTTPS 地址做备份；② 把对外一句话定位钉死成"给自闭症孩子的 AI 社交练习陪伴工具"，删掉所有"评估系统"字样；③ 准备一段 3 分钟脚本：痛点（社交练习机会少/真人陪练贵）→ 方案（低刺激 AI 陪练 + 家长观察）→ 亮点 demo（语音对话 + 超市场景 + 家长回顾）。

#### 2.6.2 商业模式：谁付费没想清，C 端评估路线与监管冲突
- **现状**：README/文档无清晰的付费方与采购路径；当前形态偏 C 端家庭，且以"评估"为卖点，与 2.1.2 的监管风险直接冲突。
- **成熟产品标准**：清晰回答"谁付费、怎么进场、客单价、续费理由"；ASD 训练在中国合规的稳妥入口是机构试点（康复中心/特校）而非 C 端诊断。
- **严重度：严重**
- **会被怎样攻击**："谁给你钱？家长为一个'非临床'的玩具持续付费的理由是什么？公立特校采购流程你走过吗？"
- **修复方向**：① 主线定为"机构试点（康复中心/特教学校）+ 治疗师 console"，付费方是机构，续费理由是纵向数据（2.5）；② 写清采购路径：公立机构走政府采购/企业服务年费，先做 1-2 家免费试点拿案例；③ C 端做免费/低价的"家庭练习版"作为引流和数据飞轮，但去掉评估定位、只讲陪伴与练习。

#### 2.6.3 护城河薄，大厂可快速复制
- **现状**：技术栈为通用拼装（DeepSeek + 腾讯云 + Three.js/VRM + MediaPipe），无自有数据、无临床合作、无独占内容；讯飞医疗/好未来类有语音、有渠道、有资质，复制门槛低。
- **成熟产品标准**：护城河来自数据闭环、临床机构背书、特教内容沉淀或合规资质之一。
- **严重度：严重**
- **会被怎样攻击**："讯飞把这套两周就做出来，还有医疗资质和渠道，你的差异化在哪？"
- **修复方向**：① 短期差异化打"ASD-friendly 体验细节 + 字面化安全话术 prompt 资产 + 家长-治疗师协作闭环"，这是大厂不愿做细的脏活；② 中期靠机构试点积累**真实标注的 ASD 社交训练对话数据**与个性化训练策略，形成数据/内容壁垒；③ 找 1 家康复机构做联合署名/共研，拿到临床背书这一最稀缺资产。

### 2.7 无障碍与体验

#### 2.7.1 感官控制粒度不足：声音/动画不能单独关，无 reduced-motion 支持
- **现状**：README 宣称"无弹跳动画/低刺激"，但实际有浮动、thinking 摇头、halo、attachment 弹入等动画；`index.css` **无 `prefers-reduced-motion`**（已 grep 确认），也无 app 内"关动画/关声音/关自动 TTS"开关。TTS 回复后自动播放，无法关闭。
- **成熟产品标准**：ASD/感官敏感用户产品应提供细粒度感官开关（动画、声音、自动播放、配色对比），并尊重系统级 `prefers-reduced-motion`。
- **严重度：严重**
- **会被怎样攻击**："你说低刺激，但孩子对声音敏感想静音对话、对动画敏感想关浮动，开关在哪？连系统的'减少动态效果'都不响应。"
- **修复方向**：① `index.css` 加 `@media (prefers-reduced-motion: reduce)` 关掉非功能动画；② 家长后台/启动页加三个开关：自动朗读、界面动画、声学监测，存 localStorage 并全局生效；③ 这些是 ASD 产品的"基本盘"，改动小、说服力大，优先做。

#### 2.7.2 运动无障碍：核心交互是"长按说话"，对运动/执行功能困难者不友好
- **现状**：录音需 `onPointerDown` 持续按住到松手（`ChatPage.jsx:639-666`），<300ms 判为"按太短"；家长入口要长按 3 秒。对手部精细动作或执行功能困难的孩子，"按住不松"本身就是门槛。
- **成熟产品标准**：提供"点一下开始/再点结束"的切换式录音替代"按住"，并允许调节超时阈值。
- **严重度：一般**
- **会被怎样攻击**："运动协调有困难的孩子按不住怎么办？只能打字？那你的语音卖点对他们就失效了。"
- **修复方向**：① 加一个"点按切换"录音模式（点开始、再点停止），作为长按的可选替代，家长设置里切换；② 把 300ms 最短时长做成可调；③ 文字输入路径已存在（兜底好），在帮助文案里说明"按不住可以打字或开点按模式"。

#### 2.7.3 可预测性档位 / 家长可配置项偏少
- **现状**：布局固定、字面化语言（做得好），但"可预测性"只有隐式默认，无显式档位；家长可配置项仅密码 + 生成报告，调不了语速、音色、话题范围、单次时长。
- **成熟产品标准**：给家长一组可配置项（语速/音量、单次时长上限、允许的话题、提示强度），让体验贴合具体孩子。
- **严重度：一般**
- **会被怎样攻击**："每个自闭症孩子敏感点不同，你的产品几乎不让家长调，怎么贴合个体？"
- **修复方向**：① 家长后台加基础设置：TTS 语速/音量（`speak` 已支持 rate/pitch/volume，接出来即可）、单次会话时长提醒（依恋护栏的 15 分钟做成可调）；② 把这些设置存 localStorage 并在 ChatPage 读用；③ 配置项不必多，3-4 个高价值项即可显著提升"个性化"叙事。

## 4. 双视角结论（两条路优先级不同，分别给）

### 4.1 比赛拿奖视角 —— 侧重 demo 外功 + 止血定位/合规雷

外功（TTS/STT/3D）已具备，**拿奖的关键不是再堆功能，而是堵住会被当场问倒的三个洞 + 把现有亮点讲好**。接下来按序补：

1. **改定位文案**（2.1.2）：删掉一切"评估系统/社交能力评估"，统一为"AI 社交练习陪伴工具"。这是评委看到的第一行，也是 Q&A 最容易翻车处，当天就改。
2. **危机文案 + 知情同意页**（2.1.3 / 2.2.1）：两个小改动，直接消解"伦理/合规"两类最狠的提问，且能在 demo 里作为"我们认真对待儿童安全"的加分点展示。
3. **可连的演示链路**（2.6.1）：香港免备案 HTTPS 或备好本机 localhost 脚本 + iOS/桌面真机各跑一遍（2.3.1），杜绝现场连不上/录不到音。
4. **纵向趋势折线 + 感官开关**（2.5.1 / 2.7.1）：低成本、高观感，把"训练有闭环""ASD 友好"从口号变成屏幕上看得见的东西。
5. **3 分钟脚本 + 评分非临床标注**（2.6.1 / 2.1.1）：主动说清边界，把"非临床"从软肋讲成"负责任"。

> 一句话：比赛阶段**不要再追新功能**，把"定位/安全/合规"的嘴堵上，把"闭环/友好"的现有料摆上桌。

### 4.2 真实落地视角 —— 侧重合规 / 临床 / 纵向，走机构试点

落地（让机构付费、敢给真孩子用）的优先级与比赛**几乎相反**，外功不重要，下面这些才决定生死：

1. **监管定位彻底转向训练辅助 + 机构试点**（2.1.2 / 2.6.2）：以"治疗师辅助记录工具"进场，由持证人员解读，绕开 C 端出分的医疗器械/广告红线。
2. **临床效度补证据链**（2.1.1）：维度映射循证框架、做信度自检、最好与 1 家康复机构联合验证——没有这个，机构专业人员不会用，更不会署名。
3. **数据合规落地**（2.2.1 / 2.2.2）：单独同意、专门处理规则、与第三方谈"不用于训练"的 DPA、本地加密 + 删除权，PIPL/未保条例逐条对齐。
4. **纵向追踪 + IEP 目标 + 治疗师 console**（2.5.1 / 2.5.2 / 2.5.3）：这是机构续费的唯一理由，必须从"单次报告"升级为"可追踪的训练系统"。
5. **危机处理做实**（2.1.3）：真实成人在场为硬前提 + 危机资源入口 + 明确"非危机干预工具"边界，这是敢给脆弱儿童用的底线。
6. **弱网鲁棒 + 场景泛化 + 多模态**（2.3.2 / 2.4.3 / 2.4.2）：适配机构差网络、覆盖谱系内低语言人群，决定产品对机构是否真有用。

> 一句话：落地阶段**外功让位于合规、临床、纵向**；先拿下一家机构的联合试点与背书，比多十个功能更重要。
