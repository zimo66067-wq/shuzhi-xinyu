# 安全漏洞上报（Security Policy）

感谢你帮助让「数智心屿」更安全。

## 报告方式

**请不要在公开 Issue 中讨论安全漏洞细节。**

发现安全问题（API 密钥泄露、未授权访问、儿童数据风险、注入漏洞等）请通过以下方式之一私下报告：

- **GitHub Security Advisory**：在仓库 → Security → "Report a vulnerability" 私密提交
- **直接联系仓库维护者**（见仓库 About 页）

## 我们关注什么

按严重程度从高到低：

1. **儿童数据泄露 / 滥用** — 任何可能导致未成年人数据被未授权访问的问题
2. **API 密钥 / 密钥泄露** — 后端 .env 或前端构建产物中的密钥泄露
3. **认证 / 授权绕过** — 家长密码、家长后台访问的绕过
4. **远程代码执行 / SSRF / 命令注入** — 尤其是 ffmpeg 调用环节
5. **Prompt 注入** — 通过对话内容绕过 ASR/对话约束
6. **XSS / CSRF** — 前端常规 Web 漏洞
7. **拒绝服务** — API 滥用、未限速

## 处置流程

1. 收到上报后 **3 个工作日**内确认
2. 评估严重性，**1-2 周**内给出修复时间承诺
3. 修复后致谢上报者（如同意公开）
4. 必要时通过 GitHub Security Advisory 发布漏洞公告

## 不在范围内

- 第三方服务（DeepSeek / 腾讯云）自身的漏洞 — 请直接向对应厂商上报
- 已部署示例站点的配置问题（请联系部署者，不是代码作者）
- 社会工程学攻击

## 安全最佳实践（部署者必读）

- ✅ 后端 `.env` **务必**在生产环境放在密钥管理服务里，而非纯文件
- ✅ 默认家长密码 `1234` **务必修改**
- ✅ 生产环境**强制 HTTPS**
- ✅ 限制 `/api/stt` / `/api/tts` / `/api/chat` 的请求速率（防止刷 API 额度）
- ✅ Nginx / 反向代理层做 IP 限流
- ✅ 日志做脱敏（不要原文记录对话内容）
- ✅ 监控异常调用模式
